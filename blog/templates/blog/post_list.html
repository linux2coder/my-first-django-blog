Every Linux user will at some point need to use the terminal or the command-line interface (CLI). It is not only for hackers & programmers; even ordinary people without any background in coding can quickly grasp the basics of the command line. So with this introduction, we shall see some basic commands which will prove to be useful for everyday tasks.

Terminal:

The terminal is an interface in which you can type and execute text-based commands. It existed before fancy desktop environments and windows systems. Now almost every desktop environment provides its own terminal-emulator, which is a GUI version of the original terminal and it usually comes with its own host of features. It is one of the most potent tools present in 'nix operating systems and with that you can do pretty much anything - given the power, you can even end up bricking your computer! So while using the terminal, you should use it with caution & respect.

If you are running Windows and are too lazy to install a terminal emulator such as MinGW or Cygwin, you can use this resource to try out these commands for yourself.

COMMANDS:

joker@joker-pc$ command [options] argument(s) #comments

This is the general form of a command. Let us examine some of the things that are visible in the code snippet. First off, "joker" is my username and "joker-pc" is my host-name. "joker@joker-pc$ " is called the prompt string and the variable 'PS1' can be used to change this to anything you want. "command" is any general command you want to execute. argument can be a filename or any other kind of information that is provided to the command for use as an input. argument is almost exactly like the argument that is passed to a function. options specify different ways for the command to process or execute. Depending on the options, the output or its format, as well as the outcome of executing the command, can be changed. Last and the least of all, comments are just ignored by the shell. Comments can be useful in annotating scripts. They start with a hashtag and whatever follows in that line gets ignored. 

joker@joker-pc$ pwd  #prints the present working directory
/home/joker

The first command we are going to see is "pwd" - it gives the full path of the present working directory. Here my pwd provides the output /home/joker which is my current directory.

joker@joker-pc$ mkdir directory_name

"mkdir test" -here the command mkdir creates a new directory called test in your present working directory.

joker@joker-pc$ cd [options] /path #(change directory)

With the help of the command cd we can change to the desired directory we want to go. "cd test" will change us from path /home/joker to /home/joker/test (check with pwd command). cd without any options will redirect you to your home directory. 

joker@joker-pc$ ls [options] argument  #(listing directory contents)

The command ls lists the contents present in your current directory. You can also specify the path of the directory whose contents you wish to see, as an argument. Now when you execute this command 

joker@joker-pc$ ls

you can see a directory called test which we created with our previous command. The ls command has got quite a few options: 

joker@joker-pc$ touch [options] file_name(s)

When used without any options, touch creates new files for all the file names that are provided as arguments (i.e., input data) provided the files with such names do not already exist. If the file already exists touch just modifies the access time and modification to the time at which the command was issued.

joker@joker-pc$ wc -lwc file_name(s)

Here the command wc is used to obtain l - no. of lines, w - word count and c - character count.

joker@joker-pc$ cat file_name(s) # in general gives the content of the file

The cat command is not only used to read the content of the file but it can even be used to concatenate the content two or more files.

joker@joker-pc$ rm [options] file_name(s) #removes that file(s)

rm is one of the most dangerous commands. rm removes the "hard-link" to the file and the actual data which is basically saying that it renders the data inaccessible (assuming that in general, there is only one hard-link to the file data). This is equivalent to deleting the file permanently.
